# Offer 捕手 — 方案说明

## 一、问题诊断

学生求职两大痛点：①**海量筛选难**——关键词匹配无法判断"金融学能否投 HR 岗"这类学科兼容问题，也无法识别企业隐性用人偏好；②**简历优化难**——现有工具只给分数不给出路，学生不知道具体差在哪、怎么改。根因是缺乏既懂岗位要求又懂候选人背景的智能匹配层。

## 二、方案设计

Offer 捕手是 AI 驱动的求职智能体，包含七个模块：

| 模块 | 功能 |
|------|------|
| 批量匹配 | 上传简历 + 添加 JD → 五维度 AI 打分排序（经验40%+学科25%+技能15%+公司10%+适配10%），权重可调 |
| 公司偏好分析 | AI 读 JD 措辞推断用人偏好（实习驱动型/院校优先型/技能导向型），分组给出策略 |
| 深度诊断 | 逐项比对 pass/warn/fail（学历/专业/经验/技能），每项附修改建议 + 预测面试题 |
| 成长路线图 | 提取目标 JD 共性要求 → 识别能力差距 → 短期/中期/长期提升建议 |
| 简历上传 | 支持拖拽 .pdf/.docx/.txt，后端自动提取文本 |
| JD 添加引导 | 粘贴后按钮呼吸动画 + Ctrl+Enter 快捷键提交 |
| 智能客服 | 右下角浮动助手，DeepSeek 驱动，引导用户使用 |

浏览器插件支持 BOSS 直聘一键抓取 JD，视觉 AI 截图识别绕过网站反爬。

## 三、AI 工具选型理由

| 模块 | 工具 | 理由 |
|------|------|------|
| 文本解析/匹配/诊断 | DeepSeek API | 中文理解强、结构化输出稳定、成本低 |
| JD 视觉提取 | 通义千问 VL (qwen-vl-plus) | 绕过 BOSS 反爬（PUA 字符、CSS 注入、DevTools 检测），比 DOM 解析可靠；plus 比 max 快 2-3 倍 |
| 智能客服 | DeepSeek API | 系统提示词注入产品知识，即问即答 |
| 文件解析 | pdfplumber + python-docx | 简历上传自动提取，支持 PDF/Word/TXT |
| 前端 | Flask + HTML/CSS/JS | 零框架依赖，极简部署 |

**为什么用视觉方案：** BOSS 直聘将薪资"13-15K"替换为 PUA 乱码字符，在文本中注入随机 CSS 规则和品牌水印，且打开 F12 即退出页面。DOM 提取需逐条对抗且维护成本高，视觉方案将截图交 VL 识别，利用语义理解区分正文与噪音，从根本上跳过反爬。

## 四、关键配置

学科七大类（商/理/工/文/医/艺/其他），AI 逐条判断不依赖死板映射。五维度权重实时显示总和，不等于 100 拦截保存。浏览器插件通过 `captureVisibleTab` 截图 → Flask → 通义千问 VL → 返回结构化 JSON，长 JD 自动多截图拼接。插件下载端点自动注入服务器地址，用户加载即用。

## 五、迭代记录与效果评估

14 个版本迭代，每次基于实际使用反馈驱动：v1.0 核心匹配 → v1.4 插件抓取 → v1.6 视觉方案（反爬对抗）→ v1.8 文件上传 → v1.10 智能客服 → v1.12 性能优化（耗时 12-18s→4-7s）→ v1.14 UI 打磨。基于 30 份真实 JD 和 5 份学生简历测试：学科兼容准确率 >90%（相比关键词提升约 40%），公司偏好分组准确率 >80%，诊断建议 90% 学生认为可执行。已部署至 Railway 公网。
