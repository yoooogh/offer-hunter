# Offer 捕手 — 方案说明

## 一、问题诊断

学生求职两大痛点：①**海量筛选难**——关键词匹配无法判断"金融学能否投 HR 岗"这类学科兼容问题，也无法识别企业隐性用人偏好；②**简历优化难**——现有工具只给分数不给出路，学生不知道具体差在哪、怎么改。根因是缺乏既懂岗位要求又懂候选人背景的智能匹配层。

## 二、方案设计

Offer 捕手是 AI 驱动的求职智能体，包含八个模块：

| 模块 | 功能 |
|------|------|
| 批量匹配 | 上传简历 + 添加 JD → 五维度 AI 打分排序（经验40%+学科25%+技能15%+公司10%+适配10%），权重可调 |
| 公司偏好分析 | AI 读 JD 措辞推断用人偏好（实习驱动型/院校优先型/技能导向型），分组给出策略 |
| 深度诊断 | 逐项比对 pass/warn/fail（学历/专业/经验/技能），每项附修改建议 + 预测面试题 |
| 成长路线图 | 提取目标 JD 共性要求 → 识别能力差距 → 短期/中期/长期提升建议 |
| 简历智能解析 | 上传 PDF/Word/TXT → 文字提取优先（pdfplumber/python-docx/XML深挖），DeepSeek 结构化输出；扫描版 PDF 自动切换 PDF.js 客户端渲染+VL 视觉兜底。结果以格式化卡片预览展示（姓名/教育/实习/项目/校园/技能） |
| 乱码清洗 | 从网站粘贴简历时自动过滤 PUA 字符、CSS 残余、hash 水印；也支持视觉清洗（Canvas渲染为图片→VL提取） |
| JD 添加引导 | 粘贴后按钮呼吸发光 + Ctrl+Enter 快捷键提交 |
| 智能客服 | 右下角浮动助手，DeepSeek 驱动，引导用户使用全部功能 |

**浏览器插件（须在 Google Chrome 中使用）：** 支持 BOSS 直聘一键/批量抓取 JD，`captureVisibleTab` 截图 → 通义千问 VL 识别 → 提取标题/公司/完整 JD 正文。长 JD 自动多截图拼接。插件下载时自动注入服务器地址，加载即用。

## 三、AI 工具选型理由

| 模块 | 工具 | 理由 |
|------|------|------|
| 文本解析/匹配/诊断 | DeepSeek API | 中文理解强、结构化输出稳定、成本低 |
| JD 视觉提取 | 通义千问 VL (qwen-vl-plus) | 绕过 BOSS 反爬（PUA 字符、CSS 注入、DevTools 检测）；plus 比 max 快 2-3 倍 |
| 简历文本解析 | DeepSeek API | 文字提取优先（准确、快速、便宜），DeepSeek JSON化输出结构化简历 |
| 简历视觉兜底 | 通义千问 VL (qwen-vl-plus) | 仅扫描版 PDF 触发——PDF.js 客户端渲染页面为图片，VL 提取文字→DeepSeek结构化 |
| 智能客服 | DeepSeek API | 系统提示词注入产品知识，即问即答 |
| 前端 | Flask + HTML/CSS/JS + PDF.js | 零框架依赖；PDF.js 客户端渲染，无需服务端 PDF 处理 |

**简历解析策略（文本优先 + VL 兜底）：** Word/TXT 走文字提取→DeepSeek（<1s，成本约 0.001 元）；PDF 先 pdfplumber 提取文字，若 >200 字走 DeepSeek，<200 字（扫描版）切换 PDF.js 客户端渲染→VL 识别。Word 三层提取兜底：段落→表格→直接挖 XML 原始 `<w:t>` 标签（覆盖 WPS 文本框）。此方案将 VL 从主力降为备用，准确率从 ~60% 提升至 ~95%，成本降至原来的 1/10。

**为什么 JD 用视觉方案：** BOSS 直聘将薪资"13-15K"替换为 PUA 乱码字符，在文本中注入随机 CSS 规则和品牌水印，且打开 F12 即退出页面。DOM 提取需逐条对抗且维护成本高，截图交 VL 识别从根本上跳过反爬。

## 四、关键配置

- **浏览器要求：插件须在 Google Chrome 中使用**（需 Manifest V3 支持及 `captureVisibleTab` API）
- 学科七大类（商/理/工/文/医/艺/其他），AI 逐条判断不依赖死板映射
- 五维度权重实时显示总和，不等于 100 拦截保存
- 简历入口：拖拽上传 PDF/Word/TXT/图片 → 自动识别。Word 三层提取（段落→表格→XML深挖），PDF 文字提取优先，扫描版自动切换 VL。手动粘贴自动清洗乱码
- JD 添加入口：文本粘贴检测触发按钮呼吸动画 + Ctrl+Enter 快捷键
- 浏览器插件：在 Chrome `chrome://extensions` 加载 `extension/` 目录；或通过网页顶部「下载浏览器插件」按钮获取自动配置好的 zip
- 分批匹配：超过 10 个 JD 自动分批，避免单次 API 调用提示词过长
- 一键部署：已部署至 Railway 公网，插件下载端点自动注入服务器地址

## 五、迭代记录与效果评估

19 个版本迭代，每次基于实际使用反馈驱动：v1.0 核心匹配 → v1.4 插件抓取 → v1.6 视觉方案（反爬对抗）→ v1.8 文件上传 → v1.10 智能客服 → v1.12 性能优化（12-18s→4-7s）→ v1.14 UI 打磨 → v2.0 VL 直出简历 → v2.1 PDF.js 客户端渲染 → v2.2 文本优先解析（DeepSeek为主+VL兜底，准确率 ~60%→~95%，成本降至 1/10）→ v2.3 Word XML 三层深挖（段落→表格→原始标签，覆盖 WPS 文本框）→ v2.4 简历格式化预览卡片（含项目/校园经历）→ v2.5 容错校验（名字缺失仍可识别）。

基于 30 份真实 JD 和 5 份学生简历测试：学科兼容准确率 >90%（相比关键词提升约 40%），公司偏好分组准确率 >80%，诊断建议 90% 学生认为可执行，简历解析可识别姓名/学校/5 段实习/项目/校园/技能等关键字段。已部署至 Railway 公网：https://offer-hunter-production.up.railway.app
